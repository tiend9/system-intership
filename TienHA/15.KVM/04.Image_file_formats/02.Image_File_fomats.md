# CÁC ĐỊNH DẠNG Ổ ĐĨA TRONG KVM

Nói đến ổ đĩa trong KVM là ta nói đến các loại định dạng **ổ đĩa ảo hoá** sau đây:

## I. ISO(International Organization for Standardization) image

### 1. Khái niệm

**ISO** bắt nguồn từ hệ thống file **ISO 9660** – Tiêu chuẩn quốc tế dành cho đĩa quang. Khi bạn **tạo một file ISO**, bạn đang thực hiện **"chụp ảnh" toàn bộ bề mặt đĩa vật lý thành một file duy nhất** có đuôi `.iso`.

### 2. Các đặc điểm cốt lõi

- **Bản sao nguyên khối**: **Một file ISO** có thể **chứa hàng nghìn file bên trong**, nhưng nó **được xử lý** như **một thực thể duy nhất**.

- **Khả năng Boot (Bootable)**: Đây là tính năng quan trọng nhất. Hầu hết **các file ISO của hệ điều hành (Windows, Linux) đều chứa một phân vùng khởi động ẩn**, **cho phép máy tính khởi động trực tiếp** từ nó khi được nạp vào USB hoặc đĩa.

- **Không nén (thông thường)**: Một **file ISO** thường có **dung lượng bằng đúng dung lượng** dữ liệu **trên đĩa gốc**. Nó **không phải** là định dạng nén như `.zip` hay `.rar`, mặc dù bạn **có thể dùng các phần mềm nén để mở nó.**

- Chỉ đọc (read-only), không thể ghi dữ liệu mới vào.(Giống đĩa **DVD quang**)

- Không phải định dạng lưu trữ chính cho máy ảo, thường dùng để **boot**.

### 3. Cách sử dụng file ISO

- **Mount (Gắn đĩa ảo)**: Bạn có thể "đánh lừa" hệ điều hành rằng có một ổ đĩa vật lý vừa được cắm vào. Trên Windows 10/11 hay Linux, bạn chỉ cần **click đúp vào file ISO để gắn nó vào một ổ đĩa ảo**.

- **Burn (Ghi đĩa)**: **Ghi file ISO** ra **đĩa CD/DVD vật lý** hoặc phổ biến hơn là **ghi ra USB Boot** (sử dụng các công cụ như **Rufus** hoặc **dd** trên **Linux**).

- **Virtualization (Ảo hóa)**: Trong các môi trường như KVM, VMware hay VirtualBox, file ISO đóng vai trò là **"đĩa cài đặt" để cài hệ điều hành cho máy ảo**.

### 4. Đặc điểm

Ưu điểm:

- **Phân phối dễ dàng**: Là cách **tiêu chuẩn** để **tải** về các bộ cài **hệ điều hành** (Ubuntu, Windows).

- **Toàn vẹn dữ liệu**: Vì là bản sao nguyên khối, **rủi ro mất file lẻ bên trong khi sao chép** là **cực thấp**.

- **Lưu trữ vĩnh viễn**: **Thay vì giữ một chồng đĩa nhựa dễ trầy xước**, bạn có thể **lưu file ISO trong ổ cứng hoặc Cloud**.

Nhược điểm:

- **Dung lượng lớn**: Một file ISO cài đặt Linux hiện nay thường từ `2GB` đến `4GB`, còn Windows có thể lên tới `5GB-6GB`.
- **Khó chỉnh sửa**: Bạn không thể đơn giản là mở file ISO ra rồi ném thêm file vào như thư mục thông thường; bạn **cần các phần mềm chuyên dụng** (như `UltraISO`) để **đóng gói lại**.

### 5. ứng dụng

- Cài đặt hệ điều hành (VD: Ubuntu ISO cho máy ảo KVM).
- Khởi động phần mềm hoặc công cụ chẩn đoán or Firewall, v.v (VD: **ISO của MemTest86** hay ISO của **Opensense**).
- Dùng để triển khai máy ảo ban đầu trong KVM.

## II. RAW

### 1. Định nghĩa

Một **file Raw image** là **một bản sao** `bit-by-bit` **chính xác tuyệt đối** của một **ổ đĩa vật lý** hoặc **một phân vùng**.

- Nó **không** có **Header**
- Nó **không** có **nén**
- Nó **không** có **metadata** của phần mềm ảo hóa.
- Cấu trúc: file Raw có các byte dữ liệu sắp xếp từ `0` đến `$N$`.

### 2. Đặc điểm

- Với **Thick Provisioning**: Nếu bạn tạo một đĩa ảo Raw 50GB, nó sẽ "ngốn" ngay lập tức 50GB trên ổ cứng vật lý của bạn.

- Với **Thin Provisioning**: dung lượng tăng dần theo dữ liệu.

- **Ưu điểm**:

  - Tốc độ: Nhanh nhất, phù hợp cho các ứng dụng cần I/O cao như Database.

  - Tính di động (Portable): Bạn có thể dùng lệnh `dd` để **đổ file Raw** này trực tiếp **vào một ổ cứng vật lý** (SSD/HDD), và **ổ cứng đó** có thể mang **cắm vào máy thật để boot bình thường**.

  - Dễ cứu hộ: Vì nó là dữ liệu thuần túy, nếu file bị lỗi, bạn **có thể dùng các công cụ khôi phục dữ liệu thông thường** (như `testdisk` hay `photorec`) để **quét trực tiếp bên trong file** như một ổ cứng thật.

- **Nhược điểm**:

  - **Không hỗ trợ Snapshot**: Không thể tự lưu các bản chụp trạng thái (Snapshot). Để làm được điều này, bạn phải dựa vào tính năng của tầng dưới (như `LVM snapshot` hoặc `ZFS snapshot`).

  - **Tốn không gian**: Khó quản lý dung lượng hơn vì nó luôn muốn chiếm hữu toàn bộ không gian được cấp.

  - **Thiếu tính năng cao cấp**: **Không hỗ trợ nén**, **không hỗ trợ mã hóa trực tiếp bên trong file** đĩa **như**định dạng `.qcow2`.

### 3. Ứng dụng

- Máy ảo cần hiệu suất tối đa (VD: cơ sở dữ liệu, máy chủ web).
- Dùng trong môi trường sản xuất yêu cầu độ tin cậy cao.
- Chuyển đổi dễ dàng sang định dạng khác hoặc sử dụng trực tiếp trên thiết bị vật lý.

## III. QCOW2 (QEMU Copy-On-Write 2)

### 1. Ý nghĩa

Định dạng ảnh đĩa do QEMU phát triển, hỗ trợ các tính năng nâng cao như **nén**, **snapshot**, và **Thin Provisioning**.

### 2. Cơ chế

- **Copy-On-Write** (`COW`) Thay vì chiếm dụng toàn bộ dung lượng ngay từ đầu, QCOW2 hoạt động theo cơ chế:

  - **Chỉ ghi những gì thay đổi**: Khi bạn tạo một ổ đĩa `100GB`, file thực tế trên host chỉ nặng vài KB (chứa header). Khi máy ảo ghi dữ liệu vào, `QCOW2` mới bắt đầu c**ấp phát các block mới** để **lưu trữ dữ liệu đó**.

  - **Cấp phát mỏng (Thin Provisioning)**: Mặc định **QCOW2 là thin-provisioned**. Điều này giúp tiết kiệm không gian đĩa vật lý cực kỳ hiệu quả.

- **Hỗ trợ Snapshot**: Bao gồm cả trạng thái của RAM, giúp bạn đóng băng máy ảo và mở lại y hệt lúc trước.

- **Backing Files**: `QCOW2` cho phép tạo các ổ đĩa dựa trên một **ổ đĩa gốc** (**Base Image**).

  - Ví dụ: Bạn có 1 file `ISO/Raw` cài sẵn Ubuntu (gọi là Base). Bạn có thể tạo 10 máy ảo từ 10 file QCOW2 khác nhau nhưng đều trỏ về Base này.

  - Dữ liệu chung nằm ở Base, 10 file kia chỉ lưu những thay đổi riêng của từng máy. Điều này giúp tạo hàng loạt máy ảo cực nhanh và tốn rất ít dung lượng.

- **Nén dữ liệu**

  - `QCOW2` hỗ trợ nén các cluster dữ liệu bên trong. Điều này rất hữu ích khi bạn muốn lưu trữ hoặc di chuyển các bản image qua mạng mà không muốn tốn nhiều băng thông.

- **Mã hóa (Encryption)**:hỗ trợ mã hóa trực tiếp trên file image (thường dùng định dạng `LUKS`), giúp bảo vệ dữ liệu bên trong máy ảo ngay cả khi file image bị đánh cắp.

### 3. Đặc điểm

**Ưu điểm**:

- **Tiết kiệm dung lượng**: File chỉ phình to khi có dữ liệu mới.
- **Cực kỳ linh hoạt**: Làm được đủ thứ từ snapshot, cloning đến nén.
- **Dễ quản lý**: Mọi thứ (snapshot, dữ liệu) nằm gọn trong 1 file duy nhất.

**Nhược điểm**:

- **Hiệu năng thấp**: Do phải qua một lớp quản lý để ánh xạ (mapping) địa chỉ và cấp phát block động, tốc độ I/O của QCOW2 thường chậm hơn Raw một chút (khoảng 5-10% tùy cấu hình).
- **Nguy cơ phân mảnh**: Do dữ liệu được ghi rải rác theo thời gian cấp phát, file QCOW2 có thể bị phân mảnh trên ổ cứng vật lý của host.
- **Dễ hỏng (Corruption)**: Vì cấu trúc file phức tạp (có header, bảng ánh xạ), nếu máy host bị mất điện đột ngột khi đang ghi, file QCOW2 có rủi ro bị lỗi cao hơn so với file Raw đơn giản.

### 4. Ứng dụng

- Môi trường phát triển/kiểm thử cần snapshot để khôi phục trạng thái.
- Máy ảo trong đám mây hoặc trung tâm dữ liệu (VD: OpenStack với KVM).
- Lưu trữ tiết kiệm, phù hợp khi chạy nhiều máy ảo trên cùng hệ thống.

## IV. SO SÁNH RAW VỚI QCOW2

| Tiêu chí | Raw | qcow2 |
|----------|-----|-------|
| **Dung lượng** | Thick: chiếm toàn bộ dung lượng. Thin: Ít linh hoạt hơn qcow2 | Thin: Chỉ chiếm dung lượng thực tế, hỗ trợ nén |
| **Hiệu năng** | Cao nhất, đọc/ghi trực tiếp | Thấp hơn, nhưng tối ưu với VirtIO |
| **snapshot** | Không hỗ trợ | Hỗ trợ, linh hoạt cho kiểm thử |
