### 1. `Bước 1`: Cài đặt Open VSwitch trên cả 2 Host

```bash
# Ubuntu/Debian
apt update && apt install -y openvswitch-switch

# CentOS/RHEL
yum install -y openvswitch
systemctl enable --now openvswitch

# Kiểm tra
sudo ovs-vsctl show
sudo ovs-appctl -t ovsdb-server ovsdb-server/list-dbs
```

![OpenVSwitch](./images/OpenVSwitch_12.png)

### 2. `Bước 2`: Tạo OVS Bridge và gán VM vào (trên cả 2 host)

```bash
# Tạo bridge br-int
sudo ovs-vsctl add-br br-int

# Xóa bridge
sudo ovs-vsctl del-br br-int

# Kiểm tra
sudo ovs-vsctl show
```

![OpenVSwitch](./images/OpenVSwitch_13.png)

Trên 2 KVM Host:

- Tạo 1 máy VM trên KVM Host (`VM1`,`VM2`)

- Chỉ định interface VM kết nối vào `br-int`. Nếu VM đã chạy, lấy tên interface là `vnet0`:

```bash
virsh domiflist VM1
```

![OpenVSwitch](./images/OpenVSwitch_15.png)

- Tắt VM đi để cấu hình

- Nếu VM dùng Linux bridge mặc định, chuyển sang OVS bridge:

```bash
# Xóa khỏi linux bridge cũ (nếu có - và để ý cái vnet cho đúng)
brctl delif virbr0 vnet0

# Gán vào OVS bridge
ovs-vsctl add-port br-int vnet0
```

Hoặc:

```bash
virsh edit VM1
```

Thay đoạn interface (như LinuxBridge) bằng:

```bash
<interface type='bridge'>
  <mac address='52:54:00:a2:28:5e'/>  
  <source bridge='br-int'/>
  <virtualport type='openvswitch'/>
  <model type='virtio'/>
</interface>
```

=> Sau đó defined + start lại VM

### 3. `Bước 3`: Cấu hình IP cho VM để làm Overlay network

Cấu hình file `netplan` để 2 VM nhận IP tĩnh sao cho **VM1** nhận IP `10.10.10.3` và **VM2** nhận IP `10.10.10.4`.

```bash
# VM1 (trên Host 1)
sudo ip addr flush dev enp1s0
sudo ip addr add 10.0.0.3/24 dev enp1s0
sudo ip link set enp1s0 up 

# VM2 (trên Host 2)
sudo ip addr flush dev enp1s0
sudo ip addr add 10.0.0.4/24 dev enp1s0
sudo ip link set enp1s0 up 
```

### 4. `Bước 4`: Tạo VXLAN Tunnel

=> Nhớ đừng chặn port `4789` bằng cách `stop` + `disable` thằng ufw

Trên Host KVM1 (remote = IP của Host 2):

```bash
ovs-vsctl add-port br-int vxlan0 -- \
  set interface vxlan0 \
  type=vxlan \
  options:remote_ip=192.168.70.140 \    # Máy chủ muốn kết nối đến
  options:key=100 \
  options:dst_port=4789
```

Trên Host KVM2 (remote = IP của Host 1):

```bash
ovs-vsctl add-port br-int vxlan0 -- \
  set interface vxlan0 \
  type=vxlan \
  options:remote_ip=192.168.70.139 \
  options:key=100 \           # Key 2 bên Host đồng bộ mới thông
  options:dst_port=4789
```

### 5. `Bước 5`: Verify và test

- Kiểm tra cấu hình OVS:

```bash
# Xem toàn bộ cấu hình bridge
ovs-vsctl show

# Kiểm tra port và type
ovs-vsctl list interface vxlan0

# Xem flow table
ovs-ofctl dump-flows br-int

# Xem MAC table đã học
ovs-appctl fdb/show br-int
```

- Kiểm tra tunnel tự động:

```bash
# Trên Host 1 — capture packet VXLAN
tcpdump -i ens33 udp port 4789 -nn

# Ping từ VM1 sang VM2 (cùng host)
ping -c 10 10.0.0.4
```

### 6. `Bước 6`: Bắt gói tin bằng TCPdump

Bắt gói tin trên `VXLAN` Tunnel:

- Ping từ **VM1** sang **VM2**:

```bash
ping 10.10.10.4
```

- Trên Host **KVM2** bắt gói tin qua `VXLAN` tunnel:

```bash
sudo ovs-tcpdump -i vxlan0 -nn icmp
```

![OpenVSwitch](./images/OpenVSwitch_16.png)

Bắt gói tin trên card `ens33` Tunnel rồi ping từ **VM1** -> **VM2**:

```bash
# Ping 10 gói tin VM1 -> VM2
ping -c 10 10.0.0.4

# Trên Host 1
sudo tcpdump -i ens33 udp port 4789
```

![OpenVSwitch](./images/OpenVSwitch_17.png)

**Nhận xét**: `VXLAN` = Layer 2 over Layer 3 (overlay)

- Bên trong: frame L2 (Ethernet)
- Bên ngoài: được encapsulate vào UDP/IP (L3) để đi qua mạng vật lý

`VXLAN` cần:

- **Underlay** (L3): 2 host phải reach được IP của nhau `192.168.70.139:4789` <=> `192.168.70.140:4789` (HOST A phải ping được HOST B, UDP `port 4789` (`VXLAN`) không bị chặn)
- **Overlay** (L2): `VXLAN` tạo mạng ảo cho VM (`10.10.10.3` <=> `10.10.10.4`)

Luồng Packet:

```bash
VM1 (ICMP 10.0.0.3 → 10.0.0.4)
   ↓
vxlan0
   ↓ (ENCAPSULATION)
UDP packet:
   SRC: 192.168.70.1
   DST: 192.168.70.2
   PORT: 4789 (VXLAN)
   Payload: Ethernet frame (ICMP 10.0.0.3 → 10.0.0.4)
   ↓
ens33 (physical NIC)
   ↓
Internet
   ↓
Host B vxlan0
   ↓
VM2 (10.0.0.4)
```

- Đổi IP khác subnet thì **ping fail** bởi `VXLAN` KHÔNG phải router: `VXLAN` chỉ làm: Extend Layer 2 (giống switch). Không làm Layer 3 routing.

-> `VXLAN` chỉ giúp 2 VM ở xa nhau giả vờ cùng switch L2, còn muốn đi subnet khác → phải có router thật.
